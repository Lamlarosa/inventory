## Keterkaitan Aplikasi Inventory dengan Mata Kuliah

**1. Pemrograman Web**
   - Keterkaitan: 
     - Aplikasi inventory berbasis web membutuhkan pengetahuan tentang HTML, CSS, JavaScript, dan PHP untuk membangun antarmuka, logika backend, dan interaksi pengguna.
     - Framework dan pustaka seperti Bootstrap untuk desain responsif serta penggunaan AJAX untuk mempercepat pencarian data tanpa perlu refresh halaman.
   - Contoh Implementasi:
     - Membuat halaman login, form input barang, dan tabel dinamis untuk menampilkan stok barang.
     - Implementasi validasi input menggunakan JavaScript untuk memberikan pengalaman pengguna yang baik.

**2. Basis Data**
   - Keterkaitan:
     - Basis data digunakan untuk menyimpan informasi barang, transaksi, pengguna, dan log aktivitas.
     - Penggunaan query SQL untuk manipulasi data seperti SELECT, INSERT, UPDATE, dan DELETE.
   - Contoh Implementasi:
     - Mendesain tabel **stock**, **users**, dan **transactions**.
     - Menggunakan relasi antar tabel untuk mengelola barang dan histori transaksi.
     - Query untuk menampilkan barang dengan stok rendah atau habis.

**3. Perancangan Antarmuka Pengguna**
   - Keterkaitan:
     - Desain antarmuka pengguna harus intuitif, responsif, dan menarik agar aplikasi mudah digunakan.
     - Menggunakan prinsip desain seperti konsistensi, kemudahan navigasi, dan penggunaan warna yang sesuai untuk fokus elemen.
   - Contoh Implementasi:
     - Dashboard dengan informasi penting seperti peringatan stok rendah dan grafik stok.
     - Form pencarian yang mudah digunakan dan tabel interaktif.
     - Responsivitas antarmuka menggunakan media query dan framework CSS.

**4. Praktik Analisis Perangkat Lunak**
   - Keterkaitan:
     - Melakukan analisis kebutuhan untuk memahami fitur apa yang dibutuhkan oleh pengguna.
     - Merancang flow diagram, use case, dan activity diagram untuk memastikan alur aplikasi sesuai dengan proses bisnis.
     - Menentukan prioritas fitur untuk diterapkan dalam metode incremental.
   - Contoh Implementasi:
     - Mengidentifikasi aktor dalam sistem, seperti admin dan pengguna biasa, untuk menentukan peran dan akses masing-masing.
     - Analisis kebutuhan sistem untuk fitur-fitur penting seperti login, transaksi barang, dan laporan.
     - Pengujian perangkat lunak untuk memastikan aplikasi bebas bug dan sesuai kebutuhan.
