
## Metode SDLC Incremental=

### **Increment 1: Dasar Aplikasi Inventory**
#### **Tujuan**
Membangun fitur utama untuk mencatat dan menampilkan data barang.

#### **Fitur**
- Login dan autentikasi pengguna.
- Halaman utama untuk menampilkan daftar stok barang.
- Fungsi tambah, edit, dan hapus data barang.
- Validasi dasar input data.

#### **Deliverable**
- Dasar aplikasi dengan manajemen barang sederhana.
- Database awal dengan tabel **users**, **stock**.

---

### **Increment 2: Manajemen Transaksi Barang**
#### **Tujuan**
Menambahkan fitur pencatatan barang masuk dan keluar.

#### **Fitur**
- Form barang masuk dan keluar dengan otomatisasi pembaruan stok.
- Histori transaksi barang (masuk dan keluar).
- Fungsi pencarian barang berdasarkan nama, kategori, atau ID.

#### **Deliverable**
- Modul transaksi barang masuk/keluar.
- Database tambahan: tabel **transactions**.

---

### **Increment 3: Monitoring dan Notifikasi**
#### **Tujuan**
Memberikan kemampuan monitoring stok barang.

#### **Fitur**
- Peringatan otomatis untuk stok rendah (<10 unit).
- Peringatan untuk barang yang habis.
- Notifikasi pop-up atau email untuk admin.

#### **Deliverable**
- Dashboard monitoring stok.
- Notifikasi otomatis terintegrasi.

---

### **Increment 4: Laporan dan Ekspor Data**
#### **Tujuan**
Menambahkan fitur laporan untuk keperluan analisis dan pelaporan.

#### **Fitur**
- Pembuatan laporan stok barang (harian, mingguan, bulanan).
- Ekspor data ke format PDF/Excel.
- Tampilan grafik untuk tren stok.

#### **Deliverable**
- Modul laporan lengkap.
- Integrasi pustaka ekspor data (e.g., PHPSpreadsheet).

---

### **Increment 5: User Management dan Keamanan**
#### **Tujuan**
Meningkatkan keamanan dan mendukung multi-pengguna.

#### **Fitur**
- Sistem peran pengguna: Admin dan staf.
- Log aktivitas pengguna.
- Proteksi data dengan hashing password dan SSL.

#### **Deliverable**
- Modul manajemen pengguna.
- Implementasi keamanan aplikasi.

---

### **Increment 6: Responsivitas dan Optimalisasi**
#### **Tujuan**
Meningkatkan pengalaman pengguna dan performa aplikasi.

#### **Fitur**
- Desain responsif untuk perangkat mobile.
- Optimalisasi query database dan kinerja aplikasi.
- Dokumentasi lengkap untuk pengembangan dan penggunaan.

#### **Deliverable**
- Aplikasi siap digunakan dengan tampilan responsif.
- Dokumentasi user guide.

---

### **Keuntungan Metode Incremental**
1. **Iteratif**: Setiap tahap memberikan hasil yang dapat digunakan dan diuji.
2. **Minim Risiko**: Masalah ditemukan lebih awal, memudahkan perbaikan.
3. **Fleksibel**: Mudah menambahkan fitur tambahan berdasarkan kebutuhan pengguna.
4. **Keterlibatan Pengguna**: Feedback dapat diperoleh di setiap increment. 

Increment ini membantu memastikan pengembangan aplikasi inventory berjalan terarah dan sesuai kebutuhan bisnis.
